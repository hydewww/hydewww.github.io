# 0198. 打家劫舍

你是一个专业的小偷，计划偷窃沿街的房屋。每间房内都藏有一定的现金，影响你偷窃的唯一制约因素就是相邻的房屋装有相互连通的防盗系统，如果两间相邻的房屋在同一晚上被小偷闯入，系统会自动报警。

给定一个代表每个房屋存放金额的非负整数数组，计算你在不触动警报装置的情况下，能够偷窃到的最高金额。

示例 1:

输入: [1,2,3,1]
输出: 4
解释: 偷窃 1 号房屋 (金额 = 1) ，然后偷窃 3 号房屋 (金额 = 3)。
     偷窃到的最高金额 = 1 + 3 = 4 。

示例 2:

输入: [2,7,9,3,1]
输出: 12
解释: 偷窃 1 号房屋 (金额 = 2), 偷窃 3 号房屋 (金额 = 9)，接着偷窃 5 号房屋 (金额 = 1)。
     偷窃到的最高金额 = 2 + 9 + 1 = 12 。

# >

动态规划。由于两个示例较为特殊（每隔一个房屋偷），于是自己想了个例子：

输入: [2,4,1,3,5]
输出: 9
解释: 偷窃 2 号房屋 (金额 = 4) ，然后偷窃 5 号房屋 (金额 = 5)。
     偷窃到的最高金额 = 4 + 5 = 9 。

假设最高金额时需要偷窃 5 号 ( 5 ) ，由于 4 号 ( 3 ) 与之相邻不可偷窃，若偷窃 3 号 ( 1 )那么必然会偷窃 1 号 ( 2 ) ，因此只需考虑在此之前是偷窃 2 号 ( 4 ) 还是 3 号 ( 1 )。设偷窃后最高金额为 F(5) ，存放金额为 N[5] ，那么 F(5) = max( F(2) , F(3) ) + N[5] 。

以此类推， F(x) = max( F(x-3), F(x-2) ) +N[x], (x>3) 。当 x<=3 时， F(1) = N[1] ，F(2) = N[2] ，F(3) = max( F(1)+N[3], F[2]) 。

在求出所有 F 值后，只需比较 F(n) 和 F(n-1) 即可得知最高金额，时间复杂度为 O(n) 。

最后，由于公式只用访问前 3 个 F 值，因此可限制 F 值存储在 Array[x%4] 中，将空间复杂度优化成 O(1)。

func rob(nums []int) int {
	switch len(nums) {
	case 0:
		return 0
	case 1:
		return nums[0]
	case 2:
		if nums[0] > nums[1] {
			return nums[0]
		}
		return nums[1]
	}
	F := [4]int{}
	F[0] = nums[0]
	F[1] = nums[1]
	F[2] = nums[0] + nums[2]
	for i := 3; i < len(nums); i++ {
		if F[(i+2)%4] >= F[(i+1)%4] {
			F[i%4] = F[(i+2)%4] + nums[i]
		} else {
			F[i%4] = F[(i+1)%4] + nums[i]
		}
	}
	if F[(len(nums)-1)%4] > F[(len(nums)-2)%4] {
		return F[(len(nums)-1)%4]
	}
	return F[(len(nums)-2)%4]
}
